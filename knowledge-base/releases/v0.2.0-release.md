- The deterministic bump rule computed `v1.0.0` (a non-empty `### Removed` section triggered "major"), but the Removed entry was an **internal cleanup** (stale review artifacts), not a breaking consumer change. The script also used the wrong baseline (`v0.0.0` from `git describe`, because the only tag pointed at a merge commit not on develop's first-parent line) while the CHANGELOG already had `[0.1.0]` released.
- Human chose **v0.2.0** (minor): the release content is predominantly bug fixes + internal refactors/docs; `v1.0.0` would carry a premature "production-ready" claim gated by `dogfood-golden-rule.md` + `public-copy.md`.
