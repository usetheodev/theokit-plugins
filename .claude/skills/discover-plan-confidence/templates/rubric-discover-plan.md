# Rubric Discover-Plan v1 — M2 Structural

This file is the **source of truth** for the M2 structural rubric for discovery PLANS produced by `/discover-plan`. It defines:

1. The 4 DAG nodes (3 hard-cap dimensions + 1 penalty-only smell dimension) with weights summing to 1.0.
2. The spec-smell detection patterns (dictionaries + regexes), shared with `discover-confidence`'s rubric-blueprint and `plan-confidence`'s rubric-v1.
3. Hard cap values for failure actions.

**v1.0 (2026-05-30):** Initial rubric, copy-with-attribution from `discover-confidence/templates/rubric-blueprint.md`. Adapted for **discovery plans** (output of `/discover-plan`), not blueprints (output of `/discover-execute`). The structural targets differ in node 3:

| Discover-confidence (blueprints) | This rubric (plans) |
|---|---|
| `blueprint_completeness` — all mandatory blueprint sections populated | `plan_completeness` — all 10 mandatory plan sections present + ADR count ≥ 2 + question budget respected + method per question |

Other nodes (research_coverage, reference_citations, structural_risk) are identical in shape to the sibling rubric.

## How to Evolve

When a new check is added (e.g., M3 citation faithfulness — does the cited path actually contain the claimed symbol?), DO NOT edit this file. Create `rubric-discover-plan-v2.md` with the additional nodes and version 2.

## YAML Definition

```yaml
version: 1
created_at: 2026-05-30
updated_at: 2026-05-30
artifact_type: discover_plan

nodes:
  - id: 1
    dimension: research_coverage
    description: "All 4 coverage corners populated: tests, deps, tools, techniques. Each corner has at least one Research Question OR an explicit <!-- DEFER-CORNER: {corner} | {reason} --> marker. (cycle-discover v1.1's 5th corner prior_art is recognized as an extra header but is NOT hard-capped by this M2 rubric — see cycle-discover.md § Downstream changes required #8.)"
    detector: research_coverage
    failure_action: hard_cap
    hard_cap_value: 49
    weight: 0.30
  - id: 2
    dimension: reference_citations
    description: "Every .claude/knowledge-base/references/{project}/{path} cited in the plan exists on disk (Path.exists() == True). No fabricated citations."
    detector: reference_citations
    failure_action: hard_cap
    hard_cap_value: 49
    weight: 0.30
  - id: 3
    dimension: plan_completeness
    description: "All 10 mandatory plan sections present + non-empty (Header, Context, Objective, In/Out-Scope, ADRs, Research Questions, Coverage Matrix, Halt-loop Checkpoints, Acceptance Criteria, Global Definition of Done); ADR count ≥ 2 in ## ADRs section; total questions ∈ [5, 10] with min 1 per corner (or DEFER-CORNER marker); method present (Fase A + Fase B) on every Q-row."
    detector: plan_completeness
    failure_action: hard_cap
    hard_cap_value: 70
    weight: 0.25
  - id: 4
    dimension: structural_risk
    description: "Spec smells below threshold (subjective adjs, weak imperatives, vague pronouns, loopholes, non-verifiable terms)"
    detector: spec_smells
    failure_action: penalty
    penalty_weights:
      subjective_adjectives: -2
      weak_imperatives: -3
      vague_pronouns: -2
      loopholes: -3
      non_verifiable: -4
    weight: 0.15

smells:
  subjective_adjectives:
    pattern_type: dictionary
    words:
      - fast
      - efficient
      - robust
      - scalable
      - optimal
      - smart
      - friendly
  weak_imperatives:
    pattern_type: regex
    pattern: '\b(should|could|may|might)\b'
  vague_pronouns:
    pattern_type: regex
    pattern: '(^|\. )(This|That|These|Those)\b'
  loopholes:
    pattern_type: dictionary
    phrases:
      - "if possible"
      - "as appropriate"
      - "when applicable"
      - "where feasible"
  non_verifiable:
    pattern_type: dictionary
    words:
      - user-friendly
      - maintainable
      - scalable
      - robust
```

## Soft caps (conservative bias)

Beyond the hard caps above, soft caps may fire when signals indicate risk:

- `soft_floor_smell_density_high` — ≥ 20 weak-imperative / loophole / vague hits in prose (cap at 89).
- `soft_floor_citation_density_low` — < 1 `.claude/knowledge-base/references/` citation per 200 words of plan prose (cap at 89).

Soft caps appear in `hard_caps_triggered` with prefix `soft_floor_` but do NOT trigger `verdict == INVALID`.

## Limitations (Known)

- **M3 future** — citation faithfulness (does the cited path actually contain the claimed behavior?) requires SAFE-adapted semantic check. M2 only verifies path existence.
- **5th corner `prior_art` (cycle-discover v1.1)** — `check_research_coverage.py` still recognizes the 4-corner v1.0 shape. Plans authored against the v1.1 template (with `prior_art` corner) pass this scorer; the extra corner is NOT hard-capped here. Human reviewers catch missing `prior_art` content until M2.1 ships.
- **EC analog** — smells inside `## Anti-patterns`, `## ADRs § Anti-pattern`, or `## Halt-loop Checkpoints` sections are counted (false positives in metalanguage). Acceptable trade-off for M2.
- **Regex `\b` boundaries** — ASCII-oriented. Implementation uses `re.UNICODE` for safety.
