# Authoritative source: rules/code-quality-golden-rule.md § 6 (Per-project tuning).
# Defaults shipped with the skill: skills/code-quality/defaults/thresholds.txt.
#
# This file sits between the locked golden rule and the per-project reality.
# When unset, the value falls back to skills/code-quality/defaults/thresholds.txt.
# Project-specific overrides go here and are CHANGELOG-tracked.
#
# Format (one entry per line):
#   KEY = VALUE
#
# Where:
#   KEY     — dot-separated detector.parameter (e.g., vulture.min_confidence)
#   VALUE   — int, float, or string; type coerced by parser
#
# Lines starting with `#` and blank lines are skipped.
# Whitespace around `=` is tolerated.
#
# Adding/changing entries:
#   - Document the project-specific motivation in a comment above the line.
#   - Add a CHANGELOG entry under `[Unreleased] § Changed`.
#   - Defaults remain in skills/code-quality/defaults/thresholds.txt for portability.
