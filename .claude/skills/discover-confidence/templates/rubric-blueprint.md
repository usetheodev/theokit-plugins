# Rubric Blueprint v1 — Discover-Confidence M2 Structural

This file is the **source of truth** for the M2 structural rubric for blueprints produced by `/discover-execute`. It defines:

1. The 4 DAG nodes (3 hard-cap dimensions + 1 penalty-only smell dimension) with weights summing to 1.0.
2. The spec-smell detection patterns (dictionaries + regexes), shared with `plan-confidence`'s rubric-v1.
3. Hard cap values for failure actions.

**v1.0 (2026-05-21):** Initial rubric, derived from `plan-confidence/templates/rubric-v1.md` but adapted for **blueprints** (not implementation plans). The structural targets differ:

| Plan-confidence dimension | Discover-confidence equivalent |
|---|---|
| Coverage Matrix 100% (gaps → tasks) | Research Coverage 100% (4 corners covered) |
| ADR completeness (alternatives in Rationale) | Reference citations all exist (no fabrication) |
| TDD in bug-fix tasks | Blueprint completeness (all mandatory sections populated) |
| Spec smells (weak imperatives, loopholes) | Same |

## How to Evolve

When a new check is added (e.g., M3 citation faithfulness — does the cited path actually contain the claimed symbol?), DO NOT edit this file. Create `rubric-blueprint-v2.md` with the additional nodes and version 2.

## YAML Definition

```yaml
version: 1
created_at: 2026-05-21
updated_at: 2026-05-21
artifact_type: blueprint

nodes:
  - id: 1
    dimension: research_coverage
    description: "All 4 coverage corners populated: Integration Tests, Dependencies, Tools, Techniques. Each corner section MUST have at least one non-placeholder subsection."
    detector: research_coverage
    failure_action: hard_cap
    hard_cap_value: 49
    weight: 0.30
  - id: 2
    dimension: reference_citations
    description: "Every .claude/knowledge-base/references/{project}/{path} cited in the blueprint exists on disk (Path.exists() == True). No fabricated citations."
    detector: reference_citations
    failure_action: hard_cap
    hard_cap_value: 49
    weight: 0.30
  - id: 3
    dimension: blueprint_completeness
    description: "All mandatory sections present and non-empty: Header (version, slug, source plan), Context, Objective, Coverage Corner 1-4 with per-project subsections, Cross-cutting Comparison, ADRs (≥1 with alternatives), Recommendations."
    detector: blueprint_completeness
    failure_action: hard_cap
    hard_cap_value: 70
    weight: 0.25
  - id: 4
    dimension: structural_risk
    description: "Spec smells below threshold (subjective adjs, weak imperatives, vague pronouns, loopholes, non-verifiable terms)"
    detector: spec_smells
    failure_action: penalty
    penalty_weights:
      subjective_adjectives: -2
      weak_imperatives: -3
      vague_pronouns: -2
      loopholes: -3
      non_verifiable: -4
    weight: 0.15

smells:
  subjective_adjectives:
    pattern_type: dictionary
    words:
      - fast
      - efficient
      - robust
      - scalable
      - optimal
      - smart
      - friendly
  weak_imperatives:
    pattern_type: regex
    pattern: '\b(should|could|may|might)\b'
  vague_pronouns:
    pattern_type: regex
    pattern: '(^|\. )(This|That|These|Those)\b'
  loopholes:
    pattern_type: dictionary
    phrases:
      - "if possible"
      - "as appropriate"
      - "when applicable"
      - "where feasible"
  non_verifiable:
    pattern_type: dictionary
    words:
      - user-friendly
      - maintainable
      - scalable
      - robust
```

## Soft caps (conservative bias)

Beyond the hard caps above, soft caps may fire when signals indicate risk:

- `soft_floor_smell_density_high` — ≥ 20 weak-imperative / loophole / vague hits in prose (cap at 89).
- `soft_floor_citation_density_low` — < 1 `.claude/knowledge-base/references/` citation per 200 words of blueprint prose (cap at 89).
- `soft_floor_per_project_asymmetry` — one in-scope project gets > 80% of blueprint content while others get < 5% each (cap at 89). **Planned for M2.1; no Python checker exists yet — this cap is not fired by M2.**

Soft caps appear in `hard_caps_triggered` with prefix `soft_floor_` but do NOT trigger `verdict == INVALID`.

## Limitations (Known)

- **M3 future** — citation faithfulness (does the cited path actually contain the claimed behavior?) requires SAFE-adapted semantic check. M2 only verifies path existence.
- **EC analog** — smells inside `## Anti-patterns` or `## Blocked questions` sections of a blueprint are counted (false positives in metalanguage). Acceptable trade-off for M2.
- **Regex `\b` boundaries** — ASCII-oriented. Implementation uses `re.UNICODE` for safety.
